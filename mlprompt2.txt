## 6) Candidate baseline rules

## 7) Feature policy

Allowed:

* ORB width
* ORB width normalized by ATR / realized vol
* pre-entry break speed
* session identity
* day-of-week / calendar flags already known before entry
* pre-entry directional context
* cross-asset concordance available before entry
* pre-entry volatility state
* instrument-specific cost state if known before entry

Forbidden:

* anything from after entry
* future high/low
* MFE/MAE
* post-exit path summaries
* session-complete labels
* future double-break
* transforms fitted on full sample

This matches your own anti-cheating rules. 

## 8) Threshold standard

Do **not** threshold on accuracy.

Threshold objective:

* primary: **post-cost EV**
* secondary: drawdown-adjusted utility
* tertiary: calibration + coverage

Use a constrained optimizer:

* maximize EV
* subject to:

  * `accept_rate >= floor`
  * `accepted_trades_per_path >= floor`
  * `holdout_uplift > 0`
