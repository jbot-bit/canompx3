## Stage-Gate (MANDATORY for production edits)

Claude Code enforces a stage-gate system on production files (`pipeline/`, `trading_app/`, `scripts/`). When Codex edits these files on the same branch, it MUST use its own stage file to avoid conflicts.

**Codex stage file:** `docs/runtime/stages/codex.md` (NOT `docs/runtime/STAGE_STATE.md` — that belongs to Claude Code).

Before editing any production file, write your stage state:
```yaml
---
task: one-line description of what you're doing
mode: IMPLEMENTATION
scope_lock:
  - path/to/file1.py
  - path/to/file2.py
blast_radius:
  - file1.py callers: list who imports it
  - tests: list affected test files
  - downstream: list consumers
updated: 2026-03-29T12:00:00Z
agent: codex
---
```

Rules:
- `blast_radius` MUST be >= 30 characters listing affected files, tests, and downstream consumers
- Core files (pipeline config, trading_app config, live/, schema) require full IMPLEMENTATION mode
- Delete your stage file when the task is complete
- NEVER write to `docs/runtime/STAGE_STATE.md` — that is Claude Code's file
- If Claude has an active stage on the same files, coordinate via `HANDOFF.md`
