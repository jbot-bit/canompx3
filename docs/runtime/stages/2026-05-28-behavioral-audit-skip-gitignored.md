# Behavioral audit must skip git-ignored files

## Why
2026-05-28: NYSE_PREOPEN Stage 1 commit blocked at pre-commit [6/8] behavioral
audit. Root cause: `scripts/tools/_bundle_auto_memory.md` (gitignored, generated
by `sync_pinecone.py`) contains "PASSED all 8 checks" in incident prose. Check 1
(`check_hardcoded_check_counts`, regex `\ball\s+\d+\s+checks\b`) matched it. The
file is `git check-ignore`-confirmed ignored and `git ls-files`-confirmed untracked.

The scanner already skips `SELF_PATH`; it just has no notion of tracked-ness.

## Fix
Add `_git_ignored(path) -> bool` using `git check-ignore -q <path>` (exit 0 =
ignored). Fail-open on any CalledProcessError other than the documented
exit-1-not-ignored, FileNotFoundError (no git), or timeout → return False (scan
it). Filter both `_python_files` and `_text_files` outputs through `not
_git_ignored(f)`.
