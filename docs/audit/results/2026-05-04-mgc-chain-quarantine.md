Date: 2026-05-04
Author: Claude Code (validator-honesty-fix worktree)
Scope: MGC strategy_validator chain rerun on 2026-05-04 + 13 pre-Phase-0
grandfathered + 67 post-Phase-0 promoted rows.

This is **not** a pooled-finding doc. It is a quarantine + provenance stamp
for one survivor and the broader research-provisional cohort.

## Scope / question

**Scope:** the 2026-05-04 MGC strategy_validator chain rerun, the 1
surviving validated_setups row, and the broader cohort of 13 pre-Phase-0
grandfathered + 67 post-Phase-0 promoted rows across MGC/MES/MNQ/GC.

**Question being answered:** does the 1 MGC survivor of the 2026-05-04
chain meet Criterion 1-13 of `pre_registered_criteria.md`, and what is
the institutional-grade provenance status of the 80 already-promoted
rows in `validated_setups` as of 2026-05-04?

This is the audit step prior to any deployment decision. It does not
ask "is this strategy profitable" -- the validator already passed it
through Phase 1-3 gates. It asks "did the discovery that produced this
row meet the locked institutional criteria for admissible search-space
declaration".

## Headline

The 2026-05-04 MGC validator chain produced **1 PASSED / 18,962 REJECTED** at
stratified BH-FDR (per-session K range 1,440 -- 3,408; aggregate 19,875).

Survivor: `MGC_CME_REOPEN_E2_RR1.0_CB1_ORB_G4`

The survivor passes Phase 3 FDR (BH-adjusted p=0.0144 in K=5,300 frozen
session pool) and clears WFE / OOS / micro-era / direction-stable gates, but
the discovery run violates Criterion 1 (no pre-reg file) and Criterion 2
(MinBTL: 19,875 trials vs strict Bailey E=1.0 bound N<=3 for 2.7yr MGC clean
horizon = ~6,625x selection-bias multiplier; vs locked op cap N<=300 = ~66x).

Per `pre_registered_criteria.md` Amendment 2.7 + Criterion 1, this row is
classified `provenance.btl_status = research_provisional`. **It is not
promoted to live trading.**

## Caveats / disconfirming / limitations

- **Single-session N**: the survivor is N=86 trades. Below Criterion 7
  deployment threshold (N >= 100). Any verdict on its OOS
  generalisability is necessarily underpowered.
- **Survivor is E2 + filter ORB_G4**: the E2 break-bar look-ahead
  registry (`feedback_e2_lookahead_drift_check_landed.md`) does NOT
  taint ORB_G filters per the canonical clearance, but a re-confirmation
  pass would be appropriate before any deployment.
- **MinBTL strict bound discrepancy**: the criteria doc reads N <= 4 for
  2.7yr MGC at strict E=1.0; the retro tool reports N <= 3. Both are
  bounds on the same continuous inequality `Ln[N] <= 1.35` -- doc
  rounds up, tool rounds down. Either way realized N=19,875 is
  unreachable.
- **`era_dependent=TRUE`**: validator already flagged this; the
  strategy is regime-gated only. C9 fail.
- **`promotion_provenance` field is unchanged**: this doc stamps the
  row's status conceptually as research-provisional; there is no
  schema-level field for that today. The status lives in this audit
  doc, not in `validated_setups.promotion_provenance` (which still
  reads `VALIDATOR_NATIVE`).
- **Disconfirming pathway**: if a Pathway B pre-reg yaml is written
  retroactively for the 2026-05-04 MGC run with theory citation +
  N <= 7 hypotheses (post-backfill), and a fresh Pathway B re-run
  promotes this same strategy, the C1 + C2 fails resolve. That is the
  only honest path to upgrade the row from research-provisional.
- **GC count anomaly**: 4 post-Phase-0 GC rows exist despite GC being
  research-only (not in `ACTIVE_ORB_INSTRUMENTS`). Worth a separate
  audit pass; out of scope here.

## Implications

- The 1 MGC survivor is **not** promoted to live trading.
- The validator's headline "1 PASSED" is correct mechanically but
  misframes severity: aggregate-K is descriptive, the per-session BH stratum
  (K=5,300 for CME_REOPEN frozen at promotion) is the actual stat. Stage 2
  fixes the headline.
- The 19,875-trial enumeration violates locked Criterion 2 by ~66x. Future
  MGC discovery MUST switch to Pathway B (testing_mode: individual) with
  N <= 7 pre-registered hypotheses (after Databento backfill to 2022-06-13)
  or N <= 4 without backfill.
- The 67 post-Phase-0 rows accumulate as research-provisional history. They
  are not re-validated by this stage and remain fully gated against
  deployment until a Pathway B pre-reg + clean re-run is performed for any
  candidate.
