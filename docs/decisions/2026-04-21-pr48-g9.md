## Kill criteria (pre-committed, for pre-reg YAML g9_kill_criteria)

### Sharpe alarm

- **Threshold (trade-level):** rolling-20-trade Sharpe < 0 for consecutive 10 trades → HALT shadow.
- **Lookback:** 20 trades (per-instrument).
- **Rationale:** under the H1 hypothesis (monotonic-up at t > 7), a rolling-20-trade Sharpe < 0 for 10 consecutive observations is a > 3-sigma deviation from expected. Not a precision alarm; a ruin-detection alarm.
- **Action:** halt shadow ingest for the breaching instrument; emit alert; require user review before resume.

### N_fires floor

- **Minimum before any kill verdict:** 50 per-instrument fresh shadow trades.
- **Minimum before any confirm verdict:** 200 per-instrument (per `feedback_oos_power_floor.md` — 80% power vs IS effect size requires larger N on a single-signal test).
- **Rationale:** under a 7-day rolling expected fire rate (estimated ~8-15 trades/week per active lane), 50 fresh trades ≈ 4-6 weeks. 200 ≈ 3-4 months. Gate matches Chan Ch 1 p.7 walk-forward discipline + feedback_oos_power_floor.md.
- **Action below floor:** return UNVERIFIED verdict, NOT DEAD.

### Shiryaev-Roberts alarm

- **Score function (per Pepelyshev-Polunchenko 2015 Eq. 17-18):**
  - f₀: null = zero-mean-R distribution (IS-calibrated at rel_vol rank = median).
  - f₁: alt = shifted-mean distribution (IS-calibrated at rel_vol rank = Q5).
  - Score: Λ_n = f₁(r_n) / f₀(r_n) — per-trade likelihood ratio of shift-in-alt.
  - SR statistic: R_n = Σ_{k=1}^{n} Π_{j=k}^{n} Λ_j.
- **Control limit:** ARL₀ target = 1000 (target ~ 3-6 month false-alarm lead time per Eq. 11 under H₀).
- **Alarm action:** emit SR alarm to `sr_monitor.py` state; log alarm timestamp + trigger index; require user review.

### Shadow observation window

- **Minimum:** 2 weeks real-time from go-live timestamp.
- **Rationale:** Chan Ch 1 p.7 walk-forward discipline + Aronson forward-test discipline.
- **Lock:** wall-clock lock from Stage 6 go-live; no early promotion to real capital, regardless of shadow performance.

### Heterogeneity veto

- **Rule:** if ≥ 1 of 3 instruments' shadow-window rank-β₁ flips sign vs pre-shadow IS direction (β₁_shadow ≤ 0 when IS β₁ > 0), halt that instrument's shadow.
- **Threshold:** at least 50 shadow trades (N_fires floor) before veto applies.
- **Rationale:** `feedback_per_lane_breakdown_required.md` + `feedback_pooled_not_lane_specific.md` — a pooled finding must remain per-instrument consistent.
- **Action:** halt the breaching instrument; emit alarm; pooled claim demoted to asset-conditional (documented in post-shadow review).
